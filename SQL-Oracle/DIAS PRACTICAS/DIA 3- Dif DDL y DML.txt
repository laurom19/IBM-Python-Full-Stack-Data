Diferencias entre distinguir sentencia de lenguaje de DEFINICION de datos (DDL) y Sentencias de lenguaje de MANIPULACION de datos(DML)
DDL (DATA DEFINITION LANGUAGE)
SE USA PARA CAMBIAR, ELIMINAR Y DEFINIR OBJETOS/ENTIDAD(TAMBIEN CONOCIDA COMO TABLA) DE UNA BASE DE DATOS.
LAS MAS COMUNES DE USAR SON 
-CREATE: SE USA PARA CREAR TABLAS Y DEFINIR SUS COLUMNAS.
-ALTER: SE USA PARA MODIFICAR TABLAS, INCLUYENDO AGREGAR Y ELIMINAR COLUMNAS,Y MODIFICAR SUS TIPOS DE DATOS
ADD
UPDATE
DELETE
-TRUNCATE: SE USA PARA ELIMIAR DATOS DE UNA TABLA, PERO NO BORRAR LA TABLA MISMA
-DROP: SE USA PARA ELIMINAR UNA TABLA.

DML (DATA MANIPULACION LANGUAGE) 
SE UTILIZA PARA LEER Y MODIFICAR DATOS DE UN TABLA.
TAMBIEN SON LLAMADAS COMO OPERACIONES CRUD -C(CREATE), READ(LEER),UPDATE(ACTUALIZAR), DROP(ELIMINAR)FILAS.
ESTAN COMPUESTA POR LAS SENTENCIAS
INSERT: INSERTA UNA O VARIAS FILAS EN UNA TABLA
SELECT: LEE O SELECIONA UNA O VARIAS TABLAS DE UNA FILA
UPDATE: ACTUALIZA UNA O VARIAS FILAS DE UNA TABLA
DELETE:QUITA UNA FILA O VARIAS DE DATOS DE UNA TABLA.

SINTAXIS:
-PARA CREAR TABLA USAMOS EL CREATE

CREATE TABLE nombre_tabla 
 (atributos: nombre, tipo de dato y con su restricción o constraints (primary key)
)
Ejemplo creando tabla PROVINCIAS con id, nombre de provincia
-id_prov char(2) va a ser un carácter con longitud fija, en este caso 2
-nombre un varchar(20) (tiene una longitud de caracteres variable).
Recordar importante que cada asignación de atributo se debe separar de otro mediante una coma
CREATE TABLE Provincias (
id_prov char(2) PRIMARY KEY NOT NULL,
nom_prov varchar(20)
)


-PARA INSERTAR DATOS VAMOS A EXPLICAR DOS METODOS, RECORDAR QUE PRIMERO SE DEBE CREAR UNA TABLA Y LUEGO PODREMOS CARGAR O POBLARLA DE DATOS USANDO LA DECLARACION INSERT


PRIMERA FORMA: PARA INSERTAR SOLAMENTE UNA FILA DE DATOS
-INSERT INTO NOMBRE_TABLA (LOS ATRIBUTOS DE LA TABLA SEPARADO POR COMA) VALUES (LOS VALORES QUE QUEREMOS INSERTAR EN CADA ATRIBUTO SEPARADO POR COMA)
SIGUIENDO EL EJEMPLO DE LA PRONVICIA PARA INSERTAR LA PROVINCIA CORDOBA QUEDARIA DE LA SIG MANERA
INSERT INTO Provincias (id_prov,nom_prov) VALUES ('cb',Cordoba)

SEGUNDA FORMA: PARA INSERTAR MULTIPLES FILAS DE DATOS
SIGUIENDO EL EJEMPLO DE LA PRONVICIA PARA INSERTAR LA PROVINCIA Y BUENOS AIRES
INSERT INTO Provincias (id_prov,nom_prov) VALUES ('cb',Cordoba) ('ba','Buenos Aires')


INSTRUCCION SELECT O SELECCIONAR
SE LLAMA CONSULTA, Y LA SALIDA QUE QUEREMOS OBTENER O VISUALIZAR AL EJECUTAR ESTA CONSULTA SE LLAMA CONJUNTO DE RESULTADOS ESPECIFICOS DE UNA TABLA O TODOS LOS RESULTADOS DE UNA TABLA. ESTA ES UNA ES UNA SENTENCIA DML.

SINTAXIS 
SELECT (TIPO DE DATOS) FROM (de) NOMBRE_TABLA
PARA SELECCIONAR TODAS LAS PROVINCIAS
SELECT * FROM PROVINCIAS
PARA SELECCIONAR UN ATRIBUTO/COLUMNA DE TABLA PROVINCIAS
SELECT nom_prov FROM PROVINCIAS
PARA ENTRAR MAS A DETALLE PODEMOS SELECCIONAR TAMBIEN FILTRANDO YA UN DATO EN ESPECIAL DE UNA COLUMNA USANDO LA CLAUSULA WHERE DONDE REQUIERE UN PREDICADO O DATO QUE QUEREMOS OBTENER EN ESPECIAL
SELECT id_prov,nom_prov
FROM PROVINCIAS
WHERE nom_prov = 'Cordoba'

UN PREDICADO ES UNA CONDICION QUE EVALUA VERDADERO, FALSO O DESCONOCIDO.
TAMBIEN LO UTILIZAMOS EN LA CONDICION DE BUSQUEDA CON LA CLAUSULA WHERE COMO USAMOS ANTERIORMENTE SI NECESITAMOS SABER CUAL ES EL ID DE LA PROVINCIA CORDOBA ES DECIR LE METEMOS UNA RESTRICCION.

UPDATE Y DELETE 
ESTAS SENTENCIAS SE UTILIZAN PARA ALTERAR Y ELIMINAR DATOS DE UNA TABLA.
EN ESTAS INSTRUCCIONES TIENE MUCHA IMPORTANCIA LA CLAUSULA WHERE.

DESPUES QUE SE CREA UNA TABLA Y SE LLENA DE DATOS, LOS DATOS PUEDEN SER ALTERADOS O ACTUALIZADOS CON LA INSTRUCCION UPDATE.
UPDATE ES UNA INSTRUCCION DE UN DML O LENGUAJE DE MANIPULACION DE DATOS.
BASANDOME EN EL EJEMPLO DE PROVINCIAS, SI LUEGO DE UN TIEMPO UNA PRONVICIA CAMBIA DE NOMBRE PUEDO ACTUALIZAR LA MISMA MEDIANTE LA INSTRUCCION 
UPDATE PROVINCIAS
SET NOM_PROV = 'CATAMARCA'
WHERE ID_PROV ='CB'
